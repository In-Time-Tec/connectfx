# Reading Guide

Start with `CONTEXT.md` for canonical language and `SPEC.md` for navigation. Read `01-product-intent.md`, `02-vocabulary-and-domain-model.md`, and `03-non-functional-requirements.md` before changing architecture. Then read the feature branch and every ADR it references.

The specification describes intended contracts. The repository is pre-implementation, so documents do not imply that a package or behavior already ships.
